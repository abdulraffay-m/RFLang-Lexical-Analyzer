## Language Overview

**RFLang** is a custom programming language designed for Assignment 01 of CS4031 Compiler Construction. The lexical analyzer (scanner) tokenizes RFLang source files and produces a stream of typed tokens for use by subsequent compiler phases.

- **Language Name:** RFLang
- **File Extension:** `.rflang`
- **Implementation:** Java (JDK 8+)
- **Two Scanners:** Manual (ManualScanner.java) + JFlex (Scanner.flex → Yylex.java)

## Output Format

Both scanners produce output in the same format:

```
<TOKEN_TYPE, "lexeme", Line: n, Col: m>
```

**Example output:**
```
<KEYWORD, "start", Line: 1, Col: 1>
<IDENTIFIER, "Count", Line: 2, Col: 1>
<ASSIGNMENT_OPERATOR, "=", Line: 2, Col: 7>
<INTEGER_LITERAL, "42", Line: 2, Col: 9>
<PUNCTUATOR, ";", Line: 2, Col: 11>
```

Each run also displays a **Statistics Report**, **Symbol Table**, and **Error Report**.

## Keywords

RFLang has 12 reserved keywords. All keywords are **case-sensitive** and **lowercase**.

| Keyword | Meaning |
|---------|---------|
| `start` | Marks the beginning of a program block |
| `finish` | Marks the end of a program block |
| `loop` | Begins a loop/iteration construct |
| `condition` | Begins a conditional (if) construct |
| `declare` | Variable declaration statement |
| `output` | Print/output statement |
| `input` | Read input from user |
| `function` | Function definition |
| `return` | Return value from a function |
| `break` | Break out of a loop |
| `continue` | Skip to next loop iteration |
| `else` | Alternative branch of a condition |

**Regex:** `(start|finish|loop|condition|declare|output|input|function|return|break|continue|else)`

## Identifiers

Identifiers in RFLang follow strict rules to distinguish them from keywords and boolean literals (which are all lowercase).

**Rules:**
- Must begin with an **uppercase letter** (A–Z)
- Followed by zero or more **lowercase letters**, **digits** (0–9), or **underscores** (`_`)
- Maximum **31 characters** total

**Regex:** `[A-Z][a-z0-9_]{0,30}`

| Example | Valid? | Reason |
|---------|--------|--------|
| `Count` | ✅ Yes | Starts with uppercase |
| `Variable_name` | ✅ Yes | Uppercase start, valid tail |
| `X` | ✅ Yes | Single uppercase letter |
| `Total_sum_2024` | ✅ Yes | Includes digits and underscore |
| `Hello_world_123` | ✅ Yes | All valid characters |
| `count` | ❌ No | Starts with lowercase |
| `2Count` | ❌ No | Starts with digit |
| `myVariable` | ❌ No | Starts with lowercase |
| `Abcdefghijklmnopqrstuvwxyz01234` | ❌ No | Exceeds 31 characters |

## Literal Formats

### Integer Literals
Whole numbers without a decimal point.

**Regex:** `[0-9]+`

| Example | Valid? |
|---------|--------|
| `42` | ✅ |
| `0` | ✅ |
| `100` | ✅ |
| `12.34` | ❌ (that's a float) |
| `1,000` | ❌ (no commas) |

> **Note:** The `+` and `-` signs are tokenized as separate `ARITHMETIC_OPERATOR` tokens. The parser handles unary signs.

### Floating-Point Literals
Numbers with a decimal point and 1–6 decimal digits, with optional scientific notation.

**Regex:** `[0-9]+\.[0-9]{1,6}([eE][+-]?[0-9]+)?`

| Example | Valid? | Notes |
|---------|--------|-------|
| `3.14` | ✅ | 2 decimal digits |
| `0.123456` | ✅ | 6 decimal digits (maximum) |
| `1.5e10` | ✅ | With exponent |
| `2.0E-3` | ✅ | Uppercase E, negative exponent |
| `99.9e+5` | ✅ | Positive exponent |
| `3.` | ❌ | No decimal digits |
| `.14` | ❌ | No integer part |
| `1.2345678` | ❌ | More than 6 decimal digits |

### String Literals
Sequences of characters enclosed in double quotes.

**Regex:** `"([^"\\\n]|\\["\\ntr])*"`

**Valid escape sequences:** `\"` `\\` `\n` `\t` `\r`

```
"hello world"
"escaped \"quote\""
"line\nbreak"
"tab\there"
""
```

### Character Literals
A single character enclosed in single quotes.

**Regex:** `'([^'\\\n]|\\['\\ntr])'`

**Valid escape sequences:** `\'` `\\` `\n` `\t` `\r`

```
'a'
'\n'
'\t'
'\\'
```

### Boolean Literals
Case-sensitive exact match only.

**Regex:** `(true|false)`

| Lexeme | Valid? |
|--------|--------|
| `true` | ✅ |
| `false` | ✅ |
| `True` | ❌ (uppercase) |
| `TRUE` | ❌ (uppercase) |

## Comment Syntax

### Single-Line Comments
Begin with `##` and extend to the end of the line.

```
## This is a single-line comment
declare Count = 42  ## inline comment after a statement
##
```

**Regex:** `##[^\n]*`

### Multi-Line Comments
Enclosed between `#*` and `*#`. Can span multiple lines.

```
#*
  This is a
  multi-line comment
*#

#* short multi-line comment *#
```

**Regex:** `#*([^*]|\*+[^*#])*\*+#`

> **Note:** Single `#` alone is a lexical error. Always use `##` or `#* *#`.

## Error Handling

The scanner detects and reports the following lexical errors:

| Error Type | Example | Description |
|------------|---------|-------------|
| `INVALID_CHARACTER` | `@`, `$`, `~` | Character not part of any token pattern |
| `INVALID_IDENTIFIER` | `count`, `myVar` | Identifier starting with lowercase |
| `INVALID_IDENTIFIER` | `Abcdefghijklmnopqrstuvwxyz01234` | Identifier exceeding 31 characters |
| `MALFORMED_FLOAT` | `1.1234567` | Float with more than 6 decimal digits |
| `UNTERMINATED_STRING` | `"hello` | String not closed before end of line |
| `UNTERMINATED_CHAR` | `'a` | Character literal not closed |
| `EMPTY_CHAR` | `''` | Empty character literal |
| `MULTI_CHAR_LITERAL` | `'ab'` | Character literal with more than one character |
| `UNCLOSED_COMMENT` | `#* not closed` | Multi-line comment never closed |
| `INVALID_CHARACTER` | `#` | Lone `#` not followed by `#` or `*` |

**Error format:**
```
[ERROR] Type: <type>, Line: <n>, Col: <m>, Lexeme: "<text>", Reason: <explanation>
```

**Error recovery:** The scanner skips the invalid token and continues scanning from the next valid token start position. All errors are collected and reported at the end.

## Sample Programs

### Sample 1 — Variable Declarations and Arithmetic

```rflang
## Sample Program 1: Variable Declarations and Arithmetic
#* Demonstrates declare, assignment, and arithmetic operators *#

start {
    declare Count = 0;
    declare Total = 100;
    declare Result = 3.14;

    Count = Count + 1;
    Total -= 10;
    Result = Total ** 2;

    Count++;
    Total--;

    output Count;
    output Total;
    output Result;
}
finish
```

### Sample 2 — Conditionals and Loops

```rflang
## Sample Program 2: Conditionals and Loops
start {
    declare X = 10;
    declare Flag = true;

    condition (X > 0) {
        output X;
        X--;
    } else {
        Flag = false;
    }

    loop (Flag == true) {
        X += 1;
        condition (X >= 10) {
            break;
        }
    }

    output Flag;
}
finish
```

### Sample 3 — Functions and String Output

```rflang
## Sample Program 3: Functions
#*
  Demonstrates function definition,
  return values, and string literals
*#

function Add(A, B) {
    return A + B;
}

function Greet(Name) {
    output "Hello, ";
    output Name;
}

start {
    declare Sum = 0;
    declare Num1 = 15;
    declare Num2 = 27;

    Sum = Add(Num1, Num2);
    output Sum;

    Greet("World");

    condition (Sum != 0) {
        output "Sum is non-zero";
    }
}
finish
```

## Pattern Matching Priority

The scanner resolves ambiguities using the following priority order (Section 3.12):

1. Multi-line comments (`#*...*#`)
2. Single-line comments (`##...`)
3. Multi-character operators (`++`, `--`, `**`, `==`, `!=`, `<=`, `>=`, `&&`, `||`, `+=`, `-=`, `*=`, `/=`)
4. Keywords
5. Boolean literals
6. Identifiers
7. Floating-point literals
8. Integer literals
9. String / character literals
10. Single-character operators
11. Punctuators
12. Whitespace (skipped)
